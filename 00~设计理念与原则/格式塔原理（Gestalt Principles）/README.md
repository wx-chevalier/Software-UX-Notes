# 格式塔原理（Gestalt Principles）核心总结

格式塔原理（Gestalt Principles）是 20 世纪初由德国心理学家韦特海默、苛勒等提出的**认知心理学理论**，核心聚焦“人类如何感知复杂世界”——强调人类对事物的感知并非孤立看待单个元素，而是倾向于将零散信息**组织成有意义、有结构的整体**，即“整体大于部分之和”（The whole is greater than the sum of its parts）。

其核心价值在于揭示了人类感知的“自动组织规律”，广泛应用于 UI/UX 设计、产品交互、视觉传达、广告营销等领域，帮助设计者通过顺应感知习惯，提升信息传递效率与用户体验。

## 一、核心原则（8 大常用原则，含通俗解释+应用场景）

| 原则名称                       | 核心感知倾向                                                                                     | 通俗例子/应用场景                                                                                                                             |
| ------------------------------ | ------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 1. 接近性（Proximity）         | 距离相近的元素会被自动归为一组，视为一个整体。                                                   | - 设计：导航菜单中相邻的按钮被认为是同一模块；表格中同一行的单元格被视为关联数据。                                                            |
| 2. 相似性（Similarity）        | 形状、颜色、大小、纹理等属性相似的元素，会被感知为同一类别或整体。                               | - 设计：UI 中相同图标样式的功能按钮（如“编辑”“删除”均用灰色图标）被认为是操作组；数据报表中同类型数据用同一种颜色标注。                       |
| 3. 连续性（Continuity）        | 人类倾向于将中断的线条、元素感知为连续的整体，而非孤立的片段。                                   | - 设计：折线图中即使数据点不连续，用户仍会自动连接成趋势线；导航流程中的步骤指示器（如“1→2→3”）引导用户感知连续流程。                         |
| 4. 闭合性（Closure）           | 面对不完整的图形/信息，人类会自动填补缺口，感知为完整的整体。                                    | - 设计：Logo 设计中用不闭合的线条构成图形（如耐克“Swoosh”标志），用户仍能识别完整形态；产品引导页中用部分遮挡的图片，用户会自动脑补完整场景。 |
| 5. 整体与部分（Figure-Ground） | 人类会自动将视觉场景分为“图形”（前景，关注的主体）和“背景”（后景，次要信息），且两者可相互转换。 | - 设计：APP 弹窗（图形）覆盖在页面内容（背景）上，用户自动聚焦弹窗；按钮文字（图形）与按钮底色（背景）对比鲜明，突出核心操作。                |
| 6. 对称性（Symmetry）          | 倾向于将对称、平衡的元素感知为稳定、和谐的整体，优先识别对称结构。                               | - 设计：产品详情页的布局左右对称（如左侧图片、右侧参数）；Logo 设计中常用轴对称/中心对称提升视觉美感与记忆点。                                |
| 7. 共同命运（Common Fate）     | 朝着同一方向运动或具有相同行为趋势的元素，会被感知为关联的整体。                                 | - 设计：下拉菜单中 hover 时同步变色的选项；数据可视化中同一系列的柱状图同步高亮，被认为是同一组数据。                                         |
| 8. 简单性（Simplicity）        | 人类倾向于用最简单、最直接的方式理解复杂信息，优先感知简洁、有序的结构，而非复杂混乱的组合。     | - 设计：产品界面避免过多装饰，用简洁的线条、清晰的层级呈现信息；操作流程简化为“三步完成”，符合用户对“简单性”的感知需求。                      |

## 二、核心价值与应用场景

### 1. 核心价值

- 顺应人类天生的感知规律，减少认知负荷：让用户无需刻意思考，就能快速理解信息逻辑与结构；
- 提升设计的一致性与逻辑性：使产品界面、信息呈现更有条理，增强用户对产品的信任感与使用效率。

### 2. 典型应用场景

- **UI/UX 设计**：页面布局、组件分组、导航设计、视觉层级划分（如用相似性统一按钮样式，用接近性分组功能模块）；
- **产品交互**：流程引导、操作反馈、数据可视化（如用连续性引导用户完成注册流程，用闭合性设计进度条）；
- **视觉传达**：Logo 设计、广告海报、信息图表（如用对称性提升视觉美感，用图形-背景对比突出核心信息）；
- **内容创作**：文章结构排版、PPT 设计（如用接近性分组段落，用简单性原则简化复杂概念的呈现）。

## 三、关键启示

格式塔原理的本质是“人类感知的组织规律”，其核心不是“设计规则”，而是“理解用户如何看世界”——设计者通过顺应这些规律，让产品/设计与用户的认知习惯同频，从而实现“自然、高效、易理解”的体验。无论是产品界面、交互流程还是信息传递，都可通过该原理优化用户的感知与决策效率。
